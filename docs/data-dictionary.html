User

Stores user account information including authentication tokens and profile data.

Column Name Data Type Constraints Description
UserID Integer (Auto-increment) Primary Key Unique identifier for the user
EmailAddress String (300) Unique, Not Null User's email address (stored in lowercase)
Token String (300) Nullable Magic link authentication token
TokenExpiry DateTime Nullable Expiry date/time for the authentication token
FirstName String (100) Nullable User's first name
LastName String (100) Nullable User's last name
CreatedBy Integer Nullable ID of user who created this record
CreatedByProcess String (100) Nullable Process/system that created this record
AccountActive Boolean Not Null Whether the user account is active
Department Integer Nullable, Foreign Key → Department.DepartmentID Department the user belongs to

Department

Stores department information including domain-based registration settings.

Column Name Data Type Constraints Description
DepartmentID Integer (Auto-increment) Primary Key Unique identifier for the department
Name String (100) Not Null Department name
Domain Text Not Null Email domain for department (e.g., "example.gov.uk")
SelfRegistrationAllowed Boolean Not Null Whether users from this domain can self-register
DepartmentLeadUserID Integer Nullable User ID of the department lead

UserRole

Stores user roles for role-based access control (Administrator, Department lead, Service Administrator).

Column Name Data Type Constraints Description
UserRoleID BigInteger (Auto-increment) Primary Key Unique identifier for the user role record
UserID Integer Nullable, Foreign Key → User.UserID User this role applies to
UserRole String (50) Nullable Role name (e.g., "Administrator", "Department lead", "Service Administrator")
Active Boolean Nullable Whether the role is currently active
Created DateTime Nullable When the role was created
CreatedBy Integer Nullable ID of user who created this role
Updated DateTime Nullable When the role was last updated
UpdatedBy Integer Nullable ID of user who last updated this role

Assessment

Core table storing assessment requests and their status. This is the main entity in the system.

Column Name Data Type Constraints Description
AssessmentID Integer (Auto-increment) Primary Key Unique identifier for the assessment
Type String (50) Nullable Assessment type (e.g., "Service Assessment", "Peer Review")
Phase String (50) Nullable Service phase (e.g., "Alpha", "Beta", "Live")
Status String (50) Nullable Assessment status (Draft, Submitted, Active, Completed, Published, Rejected)
Outcome String (10) Nullable Assessment outcome (Not rated, Green, Amber, Red)
Name String (300) Nullable Service name
Description String (750) Nullable Service description
ProjectCodeYN String (3) Nullable Whether project code exists (Yes/No)
ProjectCode String (50) Nullable Project code
StartDate DateTime Nullable Service start date
EndDateYN String (3) Nullable Whether end date exists (Yes/No)
EndDate DateTime Nullable Service end date
RequestedWeeks Text Nullable Requested assessment weeks
Portfolio String (100) Nullable Portfolio assignment
DD Integer Nullable, Foreign Key → User.UserID Delivery Director user ID
SRO Integer Nullable, Foreign Key → User.UserID Senior Responsible Owner user ID
PMYN String (3) Nullable Whether Product Manager exists (Yes/No)
PM Integer Nullable, Foreign Key → User.UserID Product Manager user ID
DMYN String (3) Nullable Whether Delivery Manager exists (Yes/No)
DM Integer Nullable, Foreign Key → User.UserID Delivery Manager user ID
CreatedBy Integer Not Null, Foreign Key → User.UserID User who created the assessment (primary contact)
CreatedDate DateTime Not Null When the assessment was created
SubmittedBy Integer Nullable, Foreign Key → User.UserID User who submitted the assessment
SubmittedDate DateTime Nullable When the assessment was submitted
AssessmentDateTime DateTime Nullable Scheduled assessment date and time
AssessmentTime String (50) Nullable Assessment time slot
SubStatusCode Integer Nullable Sub-status code for detailed status tracking
PanelComments Text Nullable Overall panel comments
PanelCommentsImprove Text Nullable Panel comments on areas for improvement
PanelCommentsComplete Boolean Nullable Whether panel comments are complete
Department Integer Nullable Department the assessment belongs to
SlackID String Nullable Slack channel ID for the assessment
FIPS_ID String (100) Nullable FIPS (Find, Interact, Publish, Share) product ID

AssessmentPanel

Links assessors to specific assessments with their assigned roles.

Column Name Data Type Constraints Description
AssessmentPanelID Integer (Auto-increment) Primary Key Unique identifier for the panel assignment
Role String (50) Nullable Assessor role (Lead assessor, Design assessor, Technical assessor, Accessibility assessor, User research assessor, Performance assessor)
UserID Integer Nullable, Foreign Key → User.UserID Assessor user ID
AssessmentID Integer Nullable, Foreign Key → Assessment.AssessmentID Assessment this panel member is assigned to
UniqueID UUID Default: gen_random_uuid() Unique identifier for the panel assignment

Stores assessor information and their capabilities.

Column Name Data Type Constraints Description
AssessorID Integer (Auto-increment) Primary Key Unique identifier for the assessor
UserID Integer Nullable User ID of the assessor
PrimaryRole String (50) Nullable Primary assessor role
Active Boolean Nullable Whether the assessor is currently active
CrossGovAssessor Boolean Nullable Whether the assessor can assess across government departments
LeadAssessor Boolean Nullable Whether the assessor can be a lead assessor
ExternalAssessor Boolean Nullable Whether the assessor is external to the department

Stores training records for assessors.

Column Name Data Type Constraints Description
AssessorTrainingID Integer (Auto-increment) Primary Key Unique identifier for the training record
UniqueID UUID Not Null, Default: gen_random_uuid() Unique identifier for the training record
UserID String Not Null Assessor user ID

ServiceStandards

Stores the 14 Service Standard points and their details.

Column Name Data Type Constraints Description
ID Integer (Auto-increment) Primary Key Unique identifier for the standard
Point Integer Not Null Standard point number (1-14)
Title String (255) Not Null Standard title
Description Text Nullable Standard description
Partial String (50) Nullable Partial assessment indicator
Assessors Text Nullable Recommended assessor types
Url Text Nullable URL to standard documentation
StandardOwner String (100) Nullable Standard owner organization
AdditionalReference String (50) Nullable Additional reference information

ServiceStandardOutcomes

Stores assessment outcomes for each Service Standard point per assessment.

Column Name Data Type Constraints Description
ID BigInteger (Auto-increment) Primary Key Unique identifier for the outcome record
AssessmentID Integer Nullable, Foreign Key → Assessment.AssessmentID Assessment this outcome belongs to
Standard Integer Nullable Service Standard point number (1-14)
Outcome String (10) Nullable Assessment outcome (Green, Amber, Red, NA)
CreatedBy Integer Nullable, Foreign Key → User.UserID User who created this outcome
Created Timestamp (with timezone) Nullable When the outcome was created

Unique Constraint: AssessmentID + Standard (one outcome per standard per assessment)

Stores actions created for standards rated Red or Amber.

Column Name Data Type Constraints Description
ActionID BigInteger (Auto-increment) Primary Key Unique identifier for the action

Comments String (1000) Nullable Action description/comments
Point Integer Nullable Service Standard point this action relates to
CreatedBy Integer Nullable User who created this action
Created DateTime Nullable When the action was created
Status String (50) Nullable Action status
AssignedTo Integer Nullable User ID the action is assigned to
UniqueID UUID Default: gen_random_uuid() Unique identifier for the action
EstimatedResolutionDate DateTime Nullable Estimated date for action resolution

AssessmentID Integer Nullable, Foreign Key → Assessment.AssessmentID Assessment this team member is assigned to
UniqueID UUID Default: gen_random_uuid() Unique identifier for the team assignment

Column Name Data Type Constraints Description

UniqueID UUID Default: gen_random_uuid() Unique identifier for the artefact
Title String (70) Nullable Artefact title
Description String (200) Nullable Artefact description
Created DateTime (precision: 7) Default: NOW() When the artefact was created
CreatedBy Integer Foreign Key → User.UserID User who created this artefact
URL Text Nullable URL to the artefact

CreatedBy Integer Nullable User who created this comment

Status String (50) Nullable Comment status

SurveyData

Stores post-assessment survey responses from service teams.

Column Name Data Type Constraints Description
id Integer (Auto-increment) Primary Key Unique identifier for the survey response
AssessmentID Integer Not Null, Foreign Key → Assessment.AssessmentID Assessment this survey relates to
UserID Integer Not Null, Foreign Key → User.UserID User who completed the survey
preAssessmentCall Integer (unsigned) Not Null Rating for pre-assessment call (1-5)
organisationOfServiceAssessment Integer (unsigned) Not Null Rating for organization of service assessment (1-5)
runningOfAssessment Integer (unsigned) Not Null Rating for running of assessment on the day (1-5)
feedbackOnLowScores Text Nullable Elaboration on items scored between 1 and 3
specificFeedbackForAssessor Text Nullable Specific feedback for the assessor panel
furtherComments Text Nullable Any further comments or suggestions

Stores audit trail of actions taken in the system.

Column Name Data Type Constraints Description
ActionID BigInteger (Auto-increment) Primary Key Unique identifier for the audit entry
AssessmentID Integer Nullable Assessment this audit entry relates to
Action String (300) Nullable Action description (e.g., "Report sent to team", "Report accepted")
Comments String (300) Nullable Additional comments
CreatedBy Integer Nullable User who performed the action
Created DateTime Nullable When the action was performed
UniqueID UUID Default: gen_random_uuid() Unique identifier for the audit entry

sessions

Stores user session data for authentication.

Column Name Data Type Constraints Description
sid Text Primary Key Session ID
sess JSON Not Null Session data in JSON format
expire Timestamp (precision: 6) Not Null, Indexed Session expiry timestamp

Database Relationships

Primary Relationships

  • User → Department: Many users belong to one department
  • User → UserRole: One user can have multiple roles
  • User → Assessor: One user can be one assessor
  • Assessment → User: Multiple relationships:
    • CreatedBy (one user created the assessment)
    • DD, SRO, PM, DM (team members)
  • Assessment → Department: Many assessments belong to one department
  • Assessment → AssessmentPanel: One assessment has many panel members
  • Assessment → ServiceStandardOutcomes: One assessment has 14 standard outcomes (one per standard)
  • Assessment → Actions: One assessment can have many actions
  • Assessment → AssessmentTeam: One assessment can have many team members
  • Assessment → AssessmentArtefacts: One assessment can have many artefacts
  • Assessment → Comments: One assessment can have many comments
  • Assessment → SurveyData: One assessment can have many survey responses
  • Assessment → Audit: One assessment can have many audit entries
