Welcome to the Service Assessment Service Documentation

The Service Assessment Service is a web application designed to facilitate the assessment of government services. It provides a digital service for users to book, manage, assess and view reports, as well as department assurance to administer assessments against the GOV Service Standard.

Documentation Overview

  • Features - Comprehensive documentation of all features in the Service Assessment Service, including booking assessments, managing services, assessing services, reporting, and administration.
  • Data Models - Detailed documentation of all data models used in the application, including their structure, relationships, and usage patterns.
  • Data Dictionary - Complete database schema documentation including all tables, columns, data types, constraints, and relationships.
  • User Flows - Detailed user flow documentation for different user types and their interactions with the system, including authentication, booking, managing, and assessing. Includes step-by-step process flows with validation, error handling, and system responses.
  • Application Models - Documentation of all application models, their methods, and how they interact with the database layer.
  • Authentication - Complete documentation of the authentication system, including magic link authentication, session management, role-based access control, and security features.
  • Roadmap - Planned features and enhancements for the Service Assessment Service.

System Overview

Technology Stack

  • Backend: Node.js with Express.js
  • Database: PostgreSQL
  • View Engine: Nunjucks
  • Authentication: Magic link via GOV.UK Notify
  • Session Management: Express-session with PostgreSQL store
  • Frontend: GOV.UK Frontend and DfE Frontend components
  • Database Migrations: Knex.js

Key Components

  • Controllers: Handle HTTP requests and responses
  • Models: Data access layer for database operations
  • Middleware: Authentication, authorization, and session management
  • Validation: Input validation using express-validator
  • Routes: URL routing and middleware configuration

User Roles

  • Standard User: Can book and manage their own assessments
  • Assessor: Can assess services against the Service Standard
  • Administrator: Can manage assessments and users
  • Department Lead: Department-level administrator
  • Service Administrator: Cross-department administrator
