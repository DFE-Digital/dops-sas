User Model (app/models/user.js)

The User model handles all user-related database operations including authentication, profile management, and user creation.

checkAndSetUserToken
checkAndSetUserToken(emailAddress: string, token: string, tokenExpiry: Date) → Promise<number>

Checks for a user by email and updates or sets their token and token expiry if they exist. If the user does not exist, creates a new user with the given details, but only if the email domain is allowed.

emailAddress: The email address of the user
token: The new token for the user (UUID)
tokenExpiry: The expiry date and time for the new token
Returns: The user's ID if successful, -1 if domain not allowed, or 0 if an error occurs
checkToken
checkToken(token: string) → Promise<object|number>

Checks the validity of a token and returns the user details if the token is valid.

token: The token to check
Returns: The user's details (UserID, EmailAddress, FirstName, LastName, Department) if the token is valid, or 0 if it is not
UpsertUserNoToken
UpsertUserNoToken(emailAddress: string, firstName: string, lastName: string, createdBy: string, createdByProcess: string, departmentID: number) → Promise<number>

Inserts or updates a user's details based on the email address. If FirstName and LastName exist, they are not updated.

emailAddress: The user's email address
firstName: The user's first name
lastName: The user's last name
createdBy: The identifier for what created the user
createdByProcess: The process by which the user was created
departmentID: The department ID
Returns: The ID of the upserted user
getBasicUserDetails
getBasicUserDetails(userId: number) → Promise<object|null>

Fetches basic details of a user based on their user ID.

userId: The unique identifier of the user
Returns: Object with UserID, EmailAddress, FirstName, LastName, Department, or null if not found
updateName
updateName(firstName: string, lastName: string, userId: number) → Promise<void>

Updates the first and last name of a user based on their user ID.

firstName: The user's new first name
lastName: The user's new last name
userId: The unique identifier of the user
updateEmail
updateEmail(emailAddress: string, userId: number) → Promise<boolean>

Updates the email address of a user based on their user ID, if the new email address does not already exist.

emailAddress: The user's new email address
userId: The unique identifier of the user
Returns: Returns true if the update was successful, false if the email address already exists
getBasicUserDetailsByEmail
getBasicUserDetailsByEmail(emailAddress: string) → Promise<object|null>

Gets user by email address.

emailAddress: The user's email address
Returns: Object with UserID, EmailAddress, FirstName, LastName, or null if not found

Assessment Model (app/models/assessmentModel.js)

The Assessment model handles all assessment-related database operations including creation, updates, queries, and status management.

AssessmentModel (Class)

A class representing an assessment with all its properties.

createAssessment
createAssessment(model: AssessmentModel, userID: number) → Promise<number>

Creates a new assessment in the database.

Returns: The AssessmentID of the newly created assessment
getAssessmentById
getAssessmentById(id: number|string) → Promise<AssessmentModel|null>

Gets an assessment by its ID, including department name.

Returns: AssessmentModel instance or null if not found
updateAssessment
updateAssessment(id: number, model: AssessmentModel, userID: number) → Promise<void>

Updates an assessment with new data.

getDraftsForUser
getDraftsForUser(userID: number) → Promise<Array>

Gets draft assessments which have been created by the signed in user.

Returns: Array of draft assessment objects
deleteAssessment
deleteAssessment(id: number) → Promise<void>

Deletes an assessment entry from the database.

getRequestsByStatus
getRequestsByStatus(status: string, department: number) → Promise<Array>

Gets assessments by status for the department the user belongs to.

Returns: Array of assessment objects
getRequestsByMixedStatus
getRequestsByMixedStatus(statuses: Array<string>) → Promise<Array>

Gets assessments by multiple statuses for departments the user is allowed to manage.

Returns: Array of assessment objects with department names
getAssessmentsUserCanAccess
getAssessmentsUserCanAccess(userID: number) → Promise<Array>

Gets assessments that the given user can access (created by, DD, PM, DM, or team member).

Returns: Array of assessment objects
getAssessmentPanelByUserID
getAssessmentPanelByUserID(userID: number) → Promise<Array>

Gets assessments the user is a panel member on.

Returns: Array of assessment objects with panel role
checkSubmitStatus
checkSubmitStatus(assessmentID: number) → Promise<object>

Checks if an assessment can be submitted (all standards rated, actions created for Red/Amber).

Returns: Object with canSubmit boolean and reason if not ready
changePrimaryContact
changePrimaryContact(assessmentID: number, userID: number) → Promise<void>

Changes the primary contact (CreatedBy) for an assessment.

createReAssessment
createReAssessment(assessmentId: number) → Promise<number>

Creates a reassessment based on an existing assessment, resetting status and outcome.

Returns: The AssessmentID of the newly created reassessment
updateFIPSID
updateFIPSID(assessmentID: number, fipsID: string) → Promise<void>

Updates the FIPS_ID for an assessment.

getAssessmentByFIPSID
getAssessmentByFIPSID(fipsID: string) → Promise<AssessmentModel|null>

Gets assessment by FIPS ID.

Returns: AssessmentModel instance or null if not found

Other Models

AssessmentPanel Model (app/models/assessmentPanel.js)

Handles assessment panel member assignments.

  • assessmentPanel(assessmentID) - Gets panel members for assessment
  • assessmentPanelExtended(assessmentID) - Gets extended panel information
  • getActiveAssessors(departmentID) - Gets active assessors for department
  • addPanelMember(assessmentID, userID, role) - Adds member to panel
  • deletePanelMember(assessmentPanelID, uniqueID) - Removes member from panel

Team Model (app/models/team.js)

Handles assessment team member assignments.

  • getTeamForAssessmentExtended(assessmentID) - Gets team members for assessment
  • addTeam(assessmentID, userID, role) - Adds team member
  • deleteTeamMemberByID(teamID, uniqueID) - Removes team member

Artefacts Model (app/models/artefacts.js)

Handles assessment artefacts (documents, links).

  • Methods for adding, removing, and listing artefacts for assessments

Actions Model (app/models/actions.js)

Handles actions created for Red/Amber rated standards.

  • Methods for creating, updating, and managing actions

Comments Model (app/models/comments.js)

Handles comments related to Service Standard points.

  • Methods for creating, updating, and managing comments

Standards Model (app/models/standards.js)

Handles Service Standard definitions.

  • getServiceStandards() - Gets all 14 Service Standards

Audit Model (app/models/audit.js)

Handles audit trail entries.

  • addAuditEntry(assessmentID, action, comments, createdBy) - Creates audit entry

Survey Model (app/models/survey.js)

Handles post-assessment survey responses.

Slack Model (app/models/slack.js)

Handles Slack integration.
